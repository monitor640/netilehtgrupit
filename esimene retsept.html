Valmistamine

Mingi randoomne jutt sellest, kuidas asju valmistama peaks. 1. Leidke omale paariline 2. Tudeng nr 1 loob omale uue repositooriumi (Tudeng nr 2 tuleb lisada kollaboraatoriks Settings -> manage access -> invite a collaborator) 3. Tudeng nr 1 loob faili conflict.md ning kirjutab sinna sisusse paar rida teksti omal valikul. Peale loomist tuleb fail ka repositooriumisse üles panna. 4. Tudeng nr 2 võtab kollaboraatoriks olemise kutse vastu (kutse tuleb meilile) 5. Tudeng nr 2 kloonib omale selle sama loodud repositooriumi. 6. Mõlemad tudengid teevad omale uue haru. Uus haru nimetatakse üldiselt selle funktsionaalsuse järgi, millega seal tegeletakse, aga hetkel võite panna harudele enda nimed. git checkout -b Loe lisaks: https://www.git-tower.com/learn/git/faq/create-branch/ 7. Mõlemad tudengid laevad oma harud üles. git push -u origin . See peaks minema ilma probleemideta. 8. Mõlemad tudengid muudavad failis conflict.md täpselt sama rida, aga erineval viisil ning teevad commiti ja pushi. 9. Tudeng üks läheb lokaalselt main branchile git checkout master ning paneb enda haru main harru (merge) git merge ( + push) 10. Nüüd proovib tudeng nr 2 teha sama.
